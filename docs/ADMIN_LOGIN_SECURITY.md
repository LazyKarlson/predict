# Безопасность админ-логина

## Реализованные меры защиты

### 1. Rate Limiting (Ограничение частоты запросов)

**Библиотека:** `express-rate-limit`

**Настройки:**
- Максимум **10 запросов** в **15 минут** с одного IP адреса
- При превышении лимита - HTTP 429 (Too Many Requests)
- Сообщение: "Слишком много попыток входа. Попробуйте позже."

### 2. Блокировка после неудачных попыток

**Механизм:**
- После **5 неудачных попыток** входа IP блокируется на **30 минут**
- Счетчик сбрасывается при успешном входе
- Блокировка хранится в памяти сервера

**Сообщение при блокировке:**
```
Ваш IP заблокирован на X минут из-за множественных неудачных попыток входа.
```

### 3. Логирование всех попыток входа

**Формат логов:**

Неудачная попытка:
```
[SECURITY] Неудачная попытка входа #1 для пользователя "admin" с IP 192.168.1.100 в 2026-02-11T10:30:00.000Z
```

Блокировка IP:
```
[SECURITY] IP 192.168.1.100 заблокирован на 30 минут после 5 неудачных попыток
```

Попытка входа с заблокированного IP:
```
[SECURITY] Попытка входа с заблокированного IP 192.168.1.100
```

Успешный вход:
```
[SECURITY] Успешный вход администратора с IP 192.168.1.100
```

## Тестирование защиты

### Тест 1: Rate Limiting

1. Откройте `/auth/admin/login`
2. Попробуйте отправить форму **11 раз подряд** (можно с любыми данными)
3. **Ожидаемый результат:** После 10-й попытки получите HTTP 429

### Тест 2: Блокировка после 5 неудачных попыток

1. Откройте `/auth/admin/login`
2. Введите **неправильный пароль** 5 раз подряд
3. **Ожидаемый результат:** 
   - После 5-й попытки увидите сообщение о блокировке на 30 минут
   - Даже с правильным паролем вход будет заблокирован
   - В консоли сервера появятся логи всех попыток

### Тест 3: Сброс счетчика при успешном входе

1. Откройте `/auth/admin/login`
2. Введите **неправильный пароль** 3 раза
3. Введите **правильный пароль**
4. **Ожидаемый результат:** 
   - Вход успешен
   - Счетчик неудачных попыток сброшен
   - В консоли лог успешного входа

### Тест 4: Логирование

1. Выполните несколько попыток входа (правильных и неправильных)
2. Проверьте консоль сервера
3. **Ожидаемый результат:** Все попытки залогированы с меткой `[SECURITY]`

## Мониторинг в production

### Просмотр логов безопасности

```bash
# Все события безопасности
npm run dev 2>&1 | grep "\[SECURITY\]"

# Только неудачные попытки
npm run dev 2>&1 | grep "Неудачная попытка входа"

# Только блокировки
npm run dev 2>&1 | grep "заблокирован"
```

### Рекомендации для production

1. **Используйте внешнюю систему логирования:**
   - Winston + файлы
   - Bunyan + Elasticsearch
   - Syslog

2. **Храните блокировки в Redis:**
   - Блокировки сохранятся при перезапуске
   - Работает в кластере серверов

3. **Настройте алерты:**
   - Уведомление при 3+ неудачных попытках
   - Уведомление при блокировке IP
   - Мониторинг частоты блокировок

4. **Анализируйте логи:**
   - Ищите паттерны атак
   - Определяйте подозрительные IP
   - Добавляйте в черный список

## Ограничения текущей реализации

### ⚠️ Хранение в памяти

**Проблема:** При перезапуске сервера:
- Все блокировки сбрасываются
- История попыток теряется

**Решение для production:** Используйте Redis или базу данных.

### ⚠️ Обход через прокси

**Проблема:** Злоумышленник может менять IP через прокси/VPN.

**Решение:** 
- Добавьте CAPTCHA после 3 неудачных попыток
- Используйте 2FA для админ-панели
- Ограничьте доступ к админ-панели по IP whitelist

### ⚠️ Логи только в консоль

**Проблема:** Логи не сохраняются, сложно анализировать.

**Решение:** Используйте Winston/Bunyan с записью в файлы.

## Пример настройки для production

### 1. Установка Winston для логирования

```bash
npm install winston
```

### 2. Настройка Redis для блокировок

```bash
npm install redis
```

### 3. Пример кода с Redis

```typescript
import { createClient } from 'redis';

const redis = createClient();
await redis.connect();

async function isIpBlocked(ip: string): Promise<boolean> {
    const blockedUntil = await redis.get(`blocked:${ip}`);
    if (!blockedUntil) return false;
    
    if (Date.now() < parseInt(blockedUntil)) {
        return true;
    }
    
    await redis.del(`blocked:${ip}`);
    return false;
}

async function recordFailedAttempt(ip: string): Promise<void> {
    const key = `attempts:${ip}`;
    const count = await redis.incr(key);
    await redis.expire(key, 1800); // 30 минут
    
    if (count >= 5) {
        const blockedUntil = Date.now() + 30 * 60 * 1000;
        await redis.set(`blocked:${ip}`, blockedUntil.toString());
        await redis.expire(`blocked:${ip}`, 1800);
    }
}
```
