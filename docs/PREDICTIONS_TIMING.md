# ⏱️ Временные ограничения для прогнозов

## Описание

Система автоматически контролирует возможность создания и редактирования прогнозов на основе времени начала матча.

## Правила

### ✅ До начала матча
- Пользователь **может создавать** новые прогнозы
- Пользователь **может редактировать** существующие прогнозы
- Форма прогноза активна и доступна
- При сохранении прогноза показывается подтверждение

### 🚫 После начала матча
- Создание новых прогнозов **запрещено**
- Редактирование существующих прогнозов **запрещено**
- Форма прогноза скрыта
- Отображается сообщение: "Матч уже начался. Прогнозы больше не принимаются."
- Если прогноз был сделан, он отображается для справки

### 🏁 После завершения матча
- Отображается результат матча
- Отображается прогноз пользователя (если был сделан)
- Отображаются заработанные баллы
- Форма прогноза недоступна

## Техническая реализация

### Backend (src/routes/api.ts)

```typescript
// Проверка времени при создании/обновлении прогноза
const matchDate = new Date(match.match_date);
const now = new Date();

if (match.is_finished) {
    return res.status(400).json({ 
        error: 'Матч уже завершен. Прогнозы больше не принимаются.' 
    });
}

if (matchDate <= now) {
    return res.status(400).json({ 
        error: 'Матч уже начался. Прогнозы больше не принимаются.' 
    });
}
```

### Frontend (views/index.ejs)

```javascript
// Определение статуса матча
const matchDate = new Date(match.match_date);
const now = new Date();
const isStarted = matchDate <= now;

// Условное отображение
if (match.is_finished) {
    // Показать результат и баллы
} else if (!isStarted) {
    // Показать форму прогноза
} else {
    // Показать сообщение о блокировке
}
```

## Визуальные индикаторы

### Цветовая кодировка карточек матчей

- **Белый фон** - матч еще не начался, можно делать прогноз
- **Желтый фон** - матч начался, прогнозы заблокированы
- **Зеленый фон** - матч завершен, результаты известны

### Сообщения для пользователя

1. **Прогноз сохранен**:
   ```
   ✓ Ваш прогноз сохранен. Вы можете изменить его до начала матча.
   ```

2. **Матч начался**:
   ```
   ⏱️ Матч уже начался. Прогнозы больше не принимаются.
   ```

3. **Не успели сделать прогноз**:
   ```
   Вы не успели сделать прогноз на этот матч
   ```

## Примеры использования

### Сценарий 1: Создание прогноза
1. Пользователь заходит на страницу матчей
2. Видит матч, который начнется через 2 часа
3. Заполняет форму прогноза
4. Нажимает "Сохранить прогноз"
5. Получает подтверждение: "✅ Прогноз успешно сохранен!"

### Сценарий 2: Редактирование прогноза
1. Пользователь заходит на страницу матчей
2. Видит свой ранее сделанный прогноз
3. Изменяет значения в форме
4. Нажимает "Обновить прогноз"
5. Получает подтверждение: "✅ Прогноз успешно сохранен!"

### Сценарий 3: Попытка прогноза после начала матча
1. Пользователь заходит на страницу матчей
2. Видит матч, который уже начался
3. Форма прогноза скрыта
4. Отображается сообщение: "⏱️ Матч уже начался. Прогнозы больше не принимаются."
5. Если прогноз был сделан ранее, он отображается для справки

### Сценарий 4: API запрос после начала матча
1. Пользователь пытается отправить прогноз через API
2. Backend проверяет время начала матча
3. Возвращает ошибку 400 с сообщением
4. Frontend показывает: "❌ Матч уже начался. Прогнозы больше не принимаются."

## Часто задаваемые вопросы

**Q: Можно ли изменить прогноз за 5 минут до начала матча?**  
A: Да, прогноз можно изменять вплоть до точного времени начала матча.

**Q: Что происходит, если я не успел сделать прогноз?**  
A: Вы не получите баллы за этот матч, но сможете продолжить делать прогнозы на другие матчи.

**Q: Можно ли удалить прогноз?**  
A: Нет, но вы можете изменить его до начала матча.

**Q: Учитывается ли часовой пояс?**
A: Да! Система полностью поддерживает часовые пояса:
- Админ вводит время в своем локальном часовом поясе
- Время автоматически конвертируется в UTC и сохраняется в БД
- Каждый пользователь видит время в своем локальном часовом поясе
- Проверка времени начала матча происходит в UTC на сервере
- Невозможно сделать ставку после начала матча из любого часового пояса

## Безопасность

- Проверка времени выполняется как на frontend, так и на backend
- Frontend проверка - для UX (скрытие формы)
- Backend проверка - для безопасности (защита от обхода)
- Невозможно обойти ограничение через API запросы
